# CS255 - DriverPass
The DriverPass project focused on developing a web-based system for a driving school. This system helps students prepare for their DMV driving tests by offering online practice exams and scheduled lessons. Key features of the platform include account creation, package purchases, lesson scheduling, progress tracking, instructor management, and administrative reporting. The overall goal was to create a secure, organized, and user-friendly platform that meets the needs of both students and the business.
I modeled system functionality using UML diagrams. The use case, activity, and class diagrams showed how users interact with the system. Workflows like scheduling lessons and completing practice tests were clearly presented. The class diagram identified the following key components: customers (students), instructors, lessons, and reservations.
If I could revise one area, I would improve the class diagram by clarifying multiplicities, removing redundancy, and better aligning attributes with methods. Enhancing modularity would also boost scalability for future expansion.
I identified user needs by focusing on the client's goals: helping students pass the DMV test and managing lesson operations effectively. This led to features like secure login, online scheduling, automated confirmations, and progress tracking. It's essential to design with user needs in mind, as even well-built systems can fail if they don't align with business expectations.
My software design approach involves analyzing requirements, identifying stakeholders, and defining system objectives. I simplify functionality into manageable components and use diagrams to visualize structure and workflows before implementation. I will continue to apply structured analysis and object-oriented design, while prioritizing scalability, security, and maintainability based on early user feedback.
